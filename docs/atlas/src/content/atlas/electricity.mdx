<Callout kind="good" title="What counts as electricity">
Löwy's receptacle — hide the volatility behind a socket you plug into. Three tests at once: ① **domain-agnostic** — no terminal/canvas/git/xterm leaks into the interface; ② **hides a hard volatility** — transport, reconnect, GPU-context loss, persistence — not a 3-line `===`; ③ **graduates** — ships as `@kolu/*` and a *different* app plugs in. The proof of ③ is real, not aspirational. [`@kolu/surface`](https://kolu.dev/blog/surface-framework/) is **kolu's own** client↔server transport first — and it has since picked up a **second consumer** in [srid/drishti](https://github.com/srid/drishti) (a fleet process-monitor; browser ↔ Bun ↔ ssh-agent) that vendors it from this repo and declares its own per-host schema with none of kolu's domain. The second consumer isn't the *point* of the extraction — it's the *proof* it was real electricity: surface never noticed the domain changed.
</Callout>

> "Decompose based on volatility." Functional decomposition splits a system by what it *does*; volatility-based decomposition isolates each area likely to *change* behind a stable interface — the way a body keeps blood-pressure, pulse, and salinity volatility "encapsulated behind the service called the heart." An *electricity*, here, is one such volatility pulled out into its own package. — Juval Löwy, [Volatility-Based Decomposition](https://www.informit.com/articles/article.aspx?p=2995357&seqNum=2) (excerpt from *Righting Software*).

## Next: extract @kolu/solid-xterm

<Callout kind="accent">
The one genuine not-yet-built electricity — mis-filed at first as "terminal domain," but a SolidJS adapter for xterm.js is agnostic (the `.agency/lowy.md` §6.5 *"what library is this?"* test). It owns two declared volatility rows — async-init cleanup (the `#598`/`#600` ~900 KB leaks) and canvas/WebGL lifecycle (the `#591` zombie-context hunt, `#575`/`#239`/`#595`).

Graduates the way surface did: any SolidJS app embedding xterm can plug in.

## Considered — not electricity

Looked framework-sized (the audit's biggest braids), but fail the bar — recorded so they don't get re-proposed.

| Candidate | Verdict | Why |
| --- | --- | --- |
| **defineMutation** (optimisticOverlay) | <Pill variant="bad">helper</Pill> | One module, one domain (canvas layouts); the write echo is already owned by `surface`'s metadata subscription. Pending is a UI cache, not a transport axis. |
| **@kolu/fs-watch** | <Pill variant="bad">helper</Pill> | Owns a refcount/debounce *lifecycle*, not a streaming transport — no wire contract, Node-bound. A `@kolu/shared` helper, not a socket. |
| **@kolu/geometry** | <Pill variant="bad">domain-coupled</Pill> | `TileLayout` binds it to a server-persisted domain type; a `Rect` package re-adapts at every boundary — moves the braid, doesn't dissolve it. |
| **@kolu/surface-publish** | <Pill variant="bad">already-covered</Pill> | Surface's `cellHandlers`/`collectionHandlers`/`pollOnEvent` already own server snapshot+delta. Only `terminal.attach` is a raw escape hatch. |
| **@kolu/commands** (palette) | <Pill variant="bad">domain-coupled</Pill> | Palette engine is already agnostic; all coupling is isolated in `createCommands`, which must stay in the app. Extracting deletes ~0 kolu lines. |
| God-procedure split | <Pill variant="run">refactor</Pill> | Its biggest dissolvable chunk *is* `solid-xterm` (above); the rest is kolu orchestration — a refactor, not a receptacle. |
